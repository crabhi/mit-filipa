# MГ­t Filipa

Party quiz game for one phone passed around: single-page app, Czech UI, no build step, no backend.

## Files

- `index.html` вЂ” the whole app (inline CSS + JS, no external dependencies).
- `kvizove_otazky.json` вЂ” 100 questions; each has `otazka` and 10 `moznosti` (`text`, `spravne`, `vysvetleni`), always exactly 5 correct.
- `sw.js` + `manifest.json` вЂ” PWA: installable, fully offline after first visit.
- `icon-*.png`, `apple-touch-icon.png` вЂ” generated by `node make_icons.mjs .` (no deps); don't edit the PNGs by hand.

## Game rules

- Setup: add players (min 1, remembered across games), choose rounds (default 3). Turns rotate round-robin.
- All players in one round get the same question; only the answer order is reshuffled per turn (so watching a previous turn doesn't map positions to picks). The deck holds one question per round.
- Player picks answers one at a time with immediate right/wrong feedback.
- Scoring ladder by correct picks: 1, 3, 6, 9, 18 points. Player may bank ("VzГ­t X bodЕЇ") anytime; 5/5 auto-banks 18.
- One wrong pick = bust: turn ends, 0 points, the wrong answer's explanation is shown in the banner.
- Never reveal unpicked answers вЂ” at turn end they dim uniformly with no truth marks or explanations.
- After all rounds: scoreboard, replay with same roster or new setup.
- The handoff screen has a small "UkonДЌit hru" link; confirming it discards the game and returns to setup (roster stays remembered).

## Tech decisions

- Questions load via `fetch('kvizove_otazky.json')` into a variable at boot вЂ” the app requires an HTTP server (`python3 -m http.server` in dev, GitHub Pages in production); `file://` is not supported. Fetch failure shows an error screen with a retry button (only reachable on a first-ever visit without network).
- Offline/PWA: `sw.js` precaches everything and serves cache-first, so installed copies boot with zero network. Requests with a `?fresh` query bypass the cache and overwrite the cached copy under the query-stripped URL. The setup screen fires `refreshQuestions()` (fire-and-forget `?fresh` fetch) to pick up question updates; it swaps in-memory `QUESTIONS` only when no game is in progress, since a live deck references the loaded pool. All paths stay relative вЂ” production is hosted under a subpath (`/mit-filipa/`). **Bump `CACHE` in `sw.js` whenever any precached file changes**, or installed apps keep serving the old version.
- All state in localStorage under `game.*` keys (in-progress game lives in `game.state`): roster, known names, rounds, used question IDs (no repeats until the pool is exhausted), and the full in-progress game (reload mid-turn в†’ "PokraДЌovat" resumes exactly, including picks).
- Layout: locked to `100dvh`, `overflow:hidden` вЂ” no screen ever scrolls. Answers are a 5Г—2 grid that flexes to fill the viewport; fonts clamp on `dvh`. Answer texts are short (max 37 chars) so two columns are safe.
- No web fonts (offline requirement) вЂ” system font stacks only. Animations are pure CSS and never block input; `prefers-reduced-motion` disables them.
- All user/data strings go through `esc()` before hitting innerHTML.

## Verifying changes

The sandbox here can't bind ports or launch browsers вЂ” for in-browser verification, ask the user to open the game themselves (or run a server via `! python3 -m http.server`). Syntax-only check: extract the inline `<script>` to a file and run `node --check` on it.
